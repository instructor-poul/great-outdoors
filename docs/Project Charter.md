# Project Charter Template

**Project Name:** Pacific Northwest Hiking Web App
**Project Sponsor:** Evan Holt

## 1. Project Purpose / Background

(Why this project is being undertaken; business need or problem it addresses)

This project seeks to create an application which provides a database for Pacific Northwest hikers that addresses issues with existing resources such as completeness of trail information, overcrowding, website functionality, and emphasis on commercial rather than environmental or safety concerns. 

## 2. Project Objectives

(What the project aims to achieve; should be specific and measurable if possible)

**Objective 1:** Furnish reliable and complete information about trails in the region and balancing trail usage

**Objective 2:** Develop the regional hiking culture by educating users on topics relating to safety and stewardship

**Objective 3:** Facilitate user feedback and coordination for group and guided hikes

### Success Criteria

Database includes all trails recognized by the following governing bodies: US Forest Service, Parks Canada at the federal level, WA State Parks and Recreation Commission, Oregon Parks and Recreation Department, BC Parks at the state/provincial level

User accounts enabled and functional with the features outlined in **Section 3**
 
## 3. Scope

In Scope:

Development of the application itself as outlined above, broken into two main categories:

- Trail Database, to include the following data: distance, elevation, seasonal attributes, difficulty (both official and user ratings), safety information, stewarship and conservation directives

- User Accounts to include the following features: trail rating and review, chat functionality, group membership, event scheduling, app feedback, donation support

- Refinement of the application based on functionality needs and stakeholder feedback

Out of Scope:

- Hosting of the application
User adoption and participation rate (number of reviews, group formation etc)

- Partnership with government or other organizations
Geographic expansion

## 9. Assumptions and Constraints

**Assumptions:**

The sources specified in Section 2 are authoritative, and can furnish the information required
Given the means to interact with the Database and with eachother, users will populate the trail entries with enough reviews and ratings to meaningfully create an information ecosystem that will draw in and engage additional users
Once the information ecosystem is sufficiently developed, park administration authorities and other groups will be amenable to partnerships with the application and its owner

**Constraints:**

The project will begin without sponsorship or assistance from park administration authorities, making it initially reliant on publically available information
Any changes to the Trail Database will rely on the foregoing, or on user input, the latter will need validation
